# VDN-агенты для RoboCup 2D Soccer

Цель: обучение многоагентной системы управления агентами с использованием обучения с подкреплением в симуляторе RoboCup Soccer Simulator 2D.

---

## Идея

В классических командах RoboCup поведение агентов задаётся вручную с помощью правил и эвристик. В данном проекте агенты обучаются самостоятельно на основе взаимодействия со средой.

Обучение выполняется в многоагентной постановке: 10 полевых игроков обучаются совместно и формируют кооперативное поведение. Агенты учатся координировать действия, принимать решения в условиях частичной наблюдаемости и эффективно взаимодействовать друг с другом.

## Запуск

### Зависимости

- Python 3 + PyTorch  
- RoboCup Soccer Simulator (rcssserver)  
- C++ (helios/sample_player)  
- Flask  
- pybind11

```

```

## Архитектура

```
rcssserver (симулятор)
    ↕
sample_player (C++) — логика агента, FSM, макро-действия
    ↕
VDNBridge
    ↕
TRAIN: HTTP (Flask-сервер)
INFERENCE: pybind11
    ↕
Python / PyTorch — VDNTrainer, Q-сети агентов, VDNMixer
```


Система работает в двух режимах:

- TRAIN (обучение)
  Агенты отправляют состояния и награды на централизованный Flask-сервер, который агрегирует опыт всех агентов и обновляет модели.

- INFERENCE (исполнение)  
  Каждый агент работает автономно, используя локальную модель без обращения к серверу.

## Пространство действий

Агент выбирает одно из 8 макро-действий:

| # | Действие | Описание |
|---|----------|----------|
| 1 | Удар по воротам | Попытка забить |
| 2 | Пас | Передача партнёру |
| 3 | Дриблинг | Обход соперника |
| 4 | Выбивание | Вынос мяча |
| 5 | Удержание | Контроль мяча |
| 6 | Перехват | Отбор мяча |
| 7 | Блокировка | Препятствие сопернику |
| 8 | Позиционирование | Выбор позиции |

Макро-действия выполняются в течение нескольких тактов симуляции.

---

## Пространство состояний

Состояние агента представляется вектором из 18 признаков:

- расстояние и угол до мяча;
- расстояние и угол до ворот соперника;
- координаты агента;
- расстояния и углы до ближайших игроков;
- параметры состояния агента (выносливость, скорость).

Формирование состояния реализовано в модуле `StateBuilder`.

---

## Обучение

Используется метод Value Decomposition Networks (VD в рамках парадигмы CTDE (Centralized Training, Decentralized Execution).

Общая функция ценности:

Q_total = sum(Q_i)

Накопленная награда (SMDP):

R_t = r_t + γ·r_{t+1} + γ²·r_{t+2} + ... + γ^τ·r_{t+τ}

---

## Функция вознаграждения

Функция вознаграждения включает:

- поощрение за приближение к цели;
- штраф за затраты выносливости;
- бонус за владение мячом;
- награду за забитый гол;
- штраф за пропущенный гол.

Параметры:

- γ = 0.99  
- w₁ = 0.04  
- w₂ = 0.001  
- бонус за владение мячом = 2.0  
- награда за гол = ±500
